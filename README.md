🔗 **[https://www.ihatepdf.cv](https://www.ihatepdf.cv)**

---

## Why browser-based PDF processing matters

Most online PDF tools — iLovePDF, Smallpdf, Adobe Acrobat online — upload 
your files to their servers. For contracts, medical records, CVs, and 
financial documents, this creates an unnecessary privacy risk.

ihatepdf.cv uses WebAssembly to run professional-grade PDF libraries 
(Ghostscript, Tesseract, pdf-lib, PDF.js) directly inside your browser tab. 
Your files are never transmitted anywhere.

**Verify it yourself:** Open DevTools → Network tab → use any tool → 
watch the upload column stay at zero.

## Tech Stack
```
pdf-lib        — PDF manipulation (merge, split, encrypt, watermark)
PDF.js         — PDF rendering (Mozilla's engine, same as Firefox)
Ghostscript    — Compression (compiled to WebAssembly)
Tesseract.js   — OCR (compiled to WebAssembly, 100+ languages)
Mammoth.js     — Word to PDF conversion
SheetJS        — Excel parsing
JSZip          — ZIP generation
jsPDF          — PDF generation from scratch
React          — UI framework
Vite           — Build tool
Service Worker — Offline caching
```

All libraries run client-side. Zero server infrastructure.

---

## Privacy Architecture
```
Your file → FileReader API → ArrayBuffer in browser memory
         → WebAssembly library (pdf-lib / Ghostscript / Tesseract)
         → Processed bytes in browser memory
         → Blob URL → download to your device

Network requests for your file data: 0
```
